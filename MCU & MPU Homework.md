# Microcontroller ກັບ Microprocessor 

## 1) ມັນແມ່ນຫຍັງ?

-  **Microprocessor (MPU)**
    - ເປັນສະໝອງຂອງລະບົບຄອມພິວເຕີ — ມັນແມ່ນ CPU (ຫນ່ວຍປະມວນຜົນກາງ) ຢູ່ໃນຊິບດຽວ
    - ບໍ່ສາມາດເຮັດວຽກຢ່າງດຽວໄດ້ — ມັນຕ້ອງການອົງປະກອບພາຍນອກເຊັ່ນ: ຫນ່ວຍຄວາມຈຳ (RAM, ROM), ຊ່ອງ input/output, ແລະ timers
    - ອອກແບບມາຕົ້ນຕໍເພື່ອປະມວນຜົນຂໍ້ມູນ ແລະໃຊ້ລະບົບປະຕິບັດການທີ່ສັບຊ້ອນ (ເຊັ່ນ: Windows, Linux, ແລະອື່ນໆ)

- **Microcontroller (MCU)**
    - ເປັນຄອມພິວເຕີນ້ອຍໆຢູ່ໃນຊິບດຽວ, ອອກແບບມາເພື່ອປະຕິບັດວຽກງານຄວບຄຸມສະເພາະ
    - ລວມມີ CPU, ຫນ່ວຍຄວາມຈຳ (RAM + ROM/Flash), ແລະຊ່ອງ I/O ທັງໝົດຢູ່ໃນຊິບດຽວ
    - ໃຊ້ຕົ້ນຕໍສຳລັບລະບົບແບບຝັງຕົວ (embedded systems) - ອຸປະກອນທີ່ປະຕິບັດວຽກງານສະເພາະ

## 2) ມັນເຮັດວຽກແນວໃດ?

- **MPU:**
    1. ດຶງຄຳສັ່ງຈາກຫນ່ວຍຄວາມຈຳພາຍນອກ (DDR)
    2. ໃຊ້ລະບົບປະຕິບັດການເຕັມຮູບແບບ (Linux/Windows/Android)
    3. ສື່ສານກັບອຸປະກອນຕ່າງໆທີ່ໄວ (PCIe/USB3/Ethernet) ຜ່ານຕົວຄວບຄຸມ
- **MCU:**
    1. ປະຕິບັດ firmware ຈາກ Flash ພາຍໃນໄປຫາ SRAM ພາຍໃນ
    2. ມັກບໍ່ມີ OS (bare-metal) ຫຼື ໃຊ້ RTOS ຂະໜາດນ້ອຍ (FreeRTOS/Zephyr)
    3. ຄວບຄຸມ pins, sensors, motors ໂດຍກົງດ້ວຍອຸປະກອນຕ່າງໆທີ່ຕິດມາກັບຊິບ

## 3) ສິ່ງທີ່ຄວນຈື່

- **ການລວມເຂົ້າກັນ**: MPU = CPU ເທົ່ານັ້ນ | MCU = CPU + ຫນ່ວຍຄວາມຈຳ + I/O ໃນຊິບດຽວ
- **ຫນ່ວຍຄວາມຈຳ**: MPU ຕ້ອງການ DDR ພາຍນອກ | MCU ມີ Flash/SRAM ພາຍໃນ
- **ພະລັງງານ**: MPU ສູງ (watts) | MCU ຕ່ຳ (µA–mA ໃນໂໝດພັກ, 10s–100s mA ໃນໂໝດເຮັດວຽກ)
- **ຊອບແວ**: MPU ໃຊ້ OS ເຕັມຮູບແບບ | MCU ໃຊ້ bare-metal/RTOS
- **ຄວາມໄວ/ປະສິດທິພາບ**: MPU ໄວກວ່າ, ຈັດການແອັບທີ່ໜັກ/UI/ML | MCU ມີຄວາມແນ່ນອນ, ດີສຳລັບການຄວບຄຸມແບບ real-time
- **ລາຄາ/BOM**: MPU ແພງກວ່າ & PCB ສັບຊ້ອນ | MCU ຖືກກວ່າ & ງ່າຍກວ່າ
- **ເວລາເປີດເຄື່ອງ**: MPU ໃຊ້ເວລາຫຼາຍວິນາທີ | MCU ໃຊ້ມິນລິວິນາທີ
- **ການນຳໃຊ້**: MPU = ໂທລະສັບ, SBCs, gateways | MCU = IoT nodes, ເຄື່ອງໃຊ້ໄຟຟ້າ, ຫຸ່ນຍົນ

## 4) ການປຽບທຽບງ່າຍໆ

- **MPU** = ສະໝອງຄອມພິວເຕີ: ສາມາດເຮັດໄດ້ຫຼາຍແຕ່ຕ້ອງການເຄື່ອງຄອມທັງໝົດຢູ່ອ້ອມຂ້າງ
- **MCU** = ສະໝອງເຄື່ອງໃຊ້: ນ້ອຍ, ປະຢັດ, ເຮັດວຽກສະເພາະຢ່າງໜ້າເຊື່ອຖື

## 5) ຕົວຢ່າງໃນໂລກຈິງ

- **ຕົວຢ່າງ MPU**: Intel Core/Ryzen, ARM Cortex-A (ເຊັ່ນ: Raspberry Pi 4/5 SoC)
- **ຕົວຢ່າງ MCU**: ESP32, STM32, ATmega328P (Arduino Uno), RP2040 (Pi Pico)

## 6) ຄວນເລືອກແບບໃດ

- **ເລືອກ MPU ຖ້າທ່ານຕ້ອງການ:**
    - Linux/Android, web browser, UI ຂະໜາດໃຫຍ່, containers, camera/display pipelines, ປະສິດທິພາບເຄືອຂ່າຍສູງ
- **ເລືອກ MCU ຖ້າທ່ານຕ້ອງການ:**
    - ໃຊ້ພະລັງງານຕ່ຳ, ຕອບສະໜອງແບບ real-time, ຄວບຄຸມ sensor/actuator ໂດຍກົງ, ຮາດແວຂະໜາດນ້ອຍ/ລາຄາຖືກ, ເປີດເຄື່ອງໄວ
- **ແບບຜະສົມ**: ໃຊ້ MCU + MPU ຮ່ວມກັນ (MCU = ຄວບຄຸມແບບ real-time, MPU = UI/cloud)

## 7) ອຸປະກອນຕ່າງໆທົ່ວໄປ (ດ້ານ MCU)

- **Digital I/O**: GPIO, PWM (motors/LED dimming)
- **ການສື່ສານ**: UART, SPI, I²C, CAN, USB (FS/HS), ບາງຄັ້ງມີ Ethernet/Wi-Fi/BLE
- **Analog**: ADC (ອ່ານ sensors), DAC (ສ້າງແຮງດັນ)
- **Timers/Counters/Watchdog** ສຳລັບການຈັບເວລາທີ່ແມ່ນຍຳ ແລະການ reset ເພື່ອຄວາມປອດໄພ

## 8) ການປຽບທຽບແບບຫຍໍ້ (ເທັກນິກຈື່ຈຳ)

- **MPU** = ປະສິດທິພາບ + OS + ອົງປະກອບພາຍນອກ
- **MCU** = ການຄວບຄຸມ + ໃຊ້ພະລັງງານຕ່ຳ + ທຸກຢ່າງໃນດຽວ
